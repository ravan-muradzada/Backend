In Laravel, a **named route** is a route that has been given a unique name using the `->name()` method. This name allows you to refer to the route **anywhere in your app**â€”like in redirects, URL generation, and Blade templatesâ€”**without hardcoding the URL**.

---

### âœ… **Why Use Named Routes?**

- Clean, maintainable, and readable code
- Decouples route paths from logic
- Easily change URLs without breaking links in your app
    
---

### ðŸ”¹ Example:

```php
Route::get('/dashboard', function () {
    return view('dashboard');
})->name('dashboard');
```

Now you can use it like this:

- In a redirect:
```php
return redirect()->route('dashboard');
```

- To generate URL:
```php

```

- In Blade:
```blade
<a href="{{ route('dashboard') }}">Dashboard</a>
```

### âœ… Named Routes with Parameters

When you name a route **with parameters**, you can still refer to it by nameâ€”just pass the parameters when generating or redirecting.

#### ðŸ”¹ Example:

```php

    return "User ID: $id";
})->name('user.profile');
```

Now you can generate a URL or redirect like this:

```php
// Generate URL

// Result: http://yourapp.com/user/5

// Redirect
return redirect()->route('user.profile', ['id' => 5]);
```
